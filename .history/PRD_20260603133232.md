## 一、产品愿景

打造一个**以时间轴为骨架、AI 为向导**的个人数字花园。它既是**求职面试时的立体简历**，也是**个人生活与知识沉淀的时光长廊**。访客可以通过**右下角 AI 导游**快速了解“我是谁”，我本人则通过**智能导入**像发朋友圈一样轻松扩充内容。

一句话：**“让面试官像翻朋友圈一样了解我，让 AI 替我开口介绍我。”**

## 二、目标用户

| 用户类型 | 核心需求 |

| **面试官 / HR** | 快速判断技术能力、项目真实性、沟通风格与成长曲线 |
| **技术同好** | 查看前端知识库链接、交流技术观点 |
| **朋友、未来的自己** | 浏览生活记录、旅行故事、时间轴上的回忆 |

## 三、功能需求

### F1. 首页
- 个人头像、一句话介绍、技能标签云。
- **三大板块入口卡片**：专业经历、成长时间轴、知识库。
- 全局导航栏 + 移动端汉堡菜单。
- 底部可加社交媒体链接与暗色模式切换。

### F2. 专业板块

#### F2.1 简历时间轴 `/resume`
- 垂直时间轴，仅展示**工作经历与项目经历**。
- 支持按年份筛选。
- 每条显示：公司/项目名、职位、起止时间、技术栈标签、摘要。
- 点击可展开详情（Markdown 渲染）。

#### F2.2 项目作品集 `/projects`
- 卡片网格展示所有项目。
- 卡片含：项目名、技术栈标签、一句话简介、链接按钮。
- 点击卡片进入 `/projects/[slug]` 详情页（Markdown 渲染，含亮点列表）。

#### F2.3 技能雷达图
- 嵌入简历页，用纯 CSS/SVG 展示技能维度。
- 数据可配置（从数据库或环境变量读取）。

#### F2.4 经历聚合页 `/experience/[slug]`
- 展示一段工作经历的基本信息（公司、职位、起止时间、技术栈、描述）
- 下方列出该经历下的所有关联项目（按时间排序）
- 每个项目以卡片形式展示，点击跳转 `/projects/[slug]`
- MVP 阶段通过时间范围自动关联，后期可支持手动关联

#### F2.5 旅行聚合页 `/travel/[slug]`（V2）
- 展示一次旅行的基本信息和按日期排列的游记卡片列表
- 每张卡片点击跳转对应游记详情

### F3. 成长时间轴 `/timeline`（朋友圈式混合流）

- **以时间为轴**，混合展示**工作、项目、学习、旅行、生活日常**等所有类型的内容。
- 每条动态像朋友圈卡片：
  - 发生时间（单点事件显示日期，时间段事件显示起止区间）。
  - 内容支持**富文本**（Markdown 渲染，支持图片、视频、音频）。
  - 类型图标与标签（💼工作、📚学习、✈️旅行、🍜生活、📝日常）。
- 可筛选类型：全部 / 工作 / 学习 / 旅行 / 生活。
- 动态从数据库读取，按事件发生时间倒序（未填则默认使用创建时间）。

### F4. 知识库 `/knowledge`
- **外部资源链接聚合页**，不内置长篇 MDX。
- 每条知识条目包含：
  - 标题
  - 外部链接（指向飞书文档等）
  - 分类标签（如 React、CSS、Next.js）
  - 简短描述
- 支持搜索与按分类筛选。
- 可手动添加或通过智能导入录入。

### F5. 全局 AI 数字分身（右下角导游机器人）

#### F5.1 形态与基础能力
- 右下角圆形悬浮按钮，点击展开聊天窗口。
- 流式对话、打字机效果、自动滚动。
- 基于脱敏个人经历回答，支持面试模拟。

#### F5.2 首次访问自动导游
- 页面加载 0.5 秒后自动展开聊天窗。
- AI 发送欢迎语：“嗨，欢迎来到何西元的数字花园！我可以带你快速了解他——你想先看看他的技术能力、项目经历，还是个人故事？”
- 附带三个快捷按钮：
  - 💼 技术能力 → 跳转 `/resume`
  - 🛠️ 项目经历 → 跳转 `/projects`
  - 📖 个人故事 → 跳转 `/timeline`
- 点击按钮后 AI 发送过渡语，然后前端完成页面跳转。

#### F5.3 页面上下文感知
- 跳转后 AI 保持展开，根据当前路由自动发送上下文引导消息：
  - 在 `/resume`：“他的工作经历涵盖电商和 SaaS 领域，要我展开讲讲某个项目吗？”
  - 在 `/projects`：“这里是他做过的项目，每个都有技术亮点，点击卡片可以看详情。”
  - 在 `/timeline`：“时间轴上不仅有工作经历，还有他的旅行故事，想看哪个？”
  - 在 `/knowledge`：“这里是他整理的前端知识库，想搜什么技术话题？”

#### F5.4 再次访问气泡策略
- 第 2-5 次访问：页面加载 30 秒后，右下角弹出气泡：“好久不见，有什么想了解的？”展示 8 秒后自动消失。不展开窗口。
- 第 6 次及以后：仅显示圆形按钮，不自动弹任何东西。
- 实现方式：`localStorage` 记录访问次数 `visitCount`。

#### F5.5 对话中智能推荐
- 当用户询问具体项目或经历，AI 回答后可附带相关链接推荐，引导跳转到对应详情页或知识库文章。

#### F5.6 问答实现策略（混合方案）
- **第一层：本地 Q&A 匹配**（免费）
  - 维护一个 Q&A 配置文件（约 30-50 条），包含关键词和预设回答。
  - 用户输入 → 关键词匹配 → 命中则直接返回，不调 API。
  - 未命中时显示统一兜底回复，引导用户浏览页面。
- **第二层：大模型兜底**（DeepSeek API，极少量调用）
  - 仅在第一层无法匹配且问题明显与个人经历相关时，才调用 API 生成回答。
  - 调用频率受每日上限控制。

#### F5.7 边界与拒绝策略
- 若问题与何西元无关（天气、股票、写代码等），AI 用幽默方式拒绝：
  - “哎呀，这个问题超出我的认知范围了，我只知道何西元的事～”
  - “你问的这个何西元没教过我，要不聊聊他的项目？”
- 始终以“何西元的分身”身份说话，不承认自己是 AI 或大模型。

#### F5.8 成本与安全
- 前端限流：同一浏览器 1 分钟内最多 5 条消息。
- 每日全站大模型调用上限：环境变量配置（默认 50 次/天）。
- 管理后台可关闭对话功能（隐身模式）。
- 系统提示词控制在 200 token 内。

### F6. 智能导入 `/import`（管理员专属）

智能导入是整个网站的核心内容入口，支持将大段文本（如完整简历、项目复盘文档、旅行游记等）自动整理为结构化内容并入库。适用于所有内容类型：工作经历、项目经历、旅行攻略、生活日常、知识库外链等。

#### F6.1 双模式导入

用户进入 `/import` 页面后，可选择两种模式：

**快速模式**
- 适用于单条内容的快速录入（如一段工作经历、一个项目简介）。
- AI 直接提取关键字段（标题、日期、技术栈、描述等），填入表单。
- 用户编辑确认后直接入库。

**深度整理模式**
- 适用于大段长文或批量内容（如完整简历、多项目复盘文档、旅行游记合集等）。
- AI 自动完成以下处理：
  1. **去重与合并**：识别并删除重复段落，合并描述同一事件的多处文字。
  2. **逻辑排序**：按时间线或主题重新组织内容，使条理清晰。
  3. **统一排版**：生成结构化的 Markdown 长文，完整保留所有真实细节（日期、数据、技术细节、过程描述）。
  4. **智能拆分**：若文本包含多个独立条目（如多个项目、多段经历），自动拆分为数组，每项独立处理。
  5. **关键字段提取**：为每条条目生成标题、一句话摘要、技术栈、亮点列表、日期等提炼信息。
- 拆分后的条目以卡片列表展示，每条包含：
  - **完整内容视图**：AI 生成的完整 Markdown 长文（用于详情页展示）。
  - **提炼字段视图**：标题、摘要、技术栈、亮点、日期等（用于列表页和卡片展示）。

#### F6.2 模板辅助

- 在输入框上方提供“模板填写”选项卡，包含工作经历、项目经历、旅行攻略等类型的格式提示。
- 用户可按模板格式粘贴内容，AI 优先匹配模板提取字段，提升准确率并降低 token 消耗。
- 模板提示与深度整理模式兼容：即使使用模板，AI 仍会进行去重和排版优化。

#### F6.3 模块选择

- 每条内容展示时，用户可勾选要导入到哪些前台模块：简历时间轴、项目作品集、成长时间轴、知识库等。
- **强制规则**：若内容类型为 `work_experience`，“简历时间轴”模块自动勾选且不可取消。
- 其余模块根据 AI 建议预选，用户可手动增删。

#### F6.4 预览编辑

- 每条内容提供“完整内容”和“提炼字段”两个可切换视图。
- 用户可修改所有字段，包括：
  - 完整 Markdown 正文
  - 标题、摘要、技术栈、亮点、日期
  - 内容类型（可在 AI 识别基础上手动更正）
  - 模块分配
- 支持手动拆分条目（若 AI 拆分不准确）或合并条目（若 AI 过度拆分）。

#### F6.5 多媒体支持

- 在编辑任意条目时，支持上传图片、视频、音频文件。
- 上传后自动在正文中插入对应的 Markdown 语法（如 `![图片描述](url)`）。
- 文件存储策略：开发期存本地 `/public/uploads`，生产环境使用 Vercel Blob 或 Cloudflare R2。
- 深度整理模式支持识别文本中已有的图片链接，自动保留并关联。

#### F6.6 状态设置

- 导入条目默认状态为“已发布”（`published`），可手动改为“草稿”（`draft`）。
- 草稿状态的内容不在前台任何页面展示。

#### F6.7 确认入库

- 用户确认后，一键批量写入数据库。每条条目的入库逻辑：
  - `Entry` 主表：存储标题、类型、状态、摘要、事件日期、内容形态等。
  - 对应类型子表（`WorkExperience` / `Project` / `Travel` 等）：存储专属字段。
  - `EntryModule` 表：按用户勾选的模块批量插入记录。
  - `TextContent` 表：存储完整 Markdown 长文（深度整理模式）或用户输入的原始文本（快速模式）。
  - `ai_analysis` 字段（Entry 表）：存储 AI 原始分析结果（含清洗前后文本对比），便于追溯。
- 使用 Prisma 事务确保原子性。
- 入库后显示结果汇总：“已添加至：简历时间轴、项目作品集、成长时间轴”，并提供跳转链接。

#### F6.8 AI 问答集成

- AI 数字分身在回答用户问题时，若命中相关条目：
  - 优先从 `TextContent` 中检索完整 Markdown 内容作为回答上下文。
  - 结合 `highlights` 等提炼字段，生成既有概括又有细节的回答。
- 这保证了 AI 的回答不是干巴巴的一句话，而是能引用项目背景、技术决策、量化结果等真实细节。

#### F6.9 权限控制

- `/import` 页面受密码保护（环境变量 + JWT 短期会话），详见 F8。
- 普通访客无感知，即使直接访问 URL 也会被密码墙拦截。

### F7. 管理后台 `/admin`（管理员专属）

- 列表展示所有条目，支持按类型筛选、搜索。
- **批量操作**：勾选多条，一键删除。
- 单条编辑：表单与导入预览一致，可修改所有字段。
- 支持切换发布/草稿状态。
- **数据导出**：一键导出全站内容为 JSON。
- **批量删除**：列表每行前有复选框，可勾选多条记录。顶部出现“已选 X 条”操作栏，包含“批量删除”按钮。点击后弹出二次确认框：“确定删除选中的 X 条记录？此操作不可恢复。”确认后调用 Server Action，使用 Prisma 事务批量级联删除关联的 TextContent、EntryModule、类型子表记录。操作完成后列表自动刷新并显示“已删除 X 条记录”提示。

#### F8. 权限控制

**方案一：密码弹窗（默认）**
- `/import` 与 `/admin` 受密码保护。
- 访问时弹出密码输入框（仅密码字段），前端发送 `POST /api/verify`。
- 后端校验环境变量 `IMPORT_PASSWORD`，匹配后签发 JWT（24 小时有效），前端存入 `localStorage`。
- 后续所有请求在 `Authorization: Bearer <token>` 头部携带，后端中间件验证。

**方案二：URL 令牌直通（可选增强）**
- 环境变量额外配置 `ADMIN_TOKEN`（一长串随机字符串）。
- 管理员可通过 `你的域名.com/admin?token=xxxx` 直接进入后台，无需输入密码。
- 后端校验 URL 参数与 `ADMIN_TOKEN` 匹配后，自动签发 JWT 并重定向到干净 URL。
- 进入后台后，导航栏自动显示所有管理入口（`/import`、`/admin`），普通访客看不到这些链接。

**共同约束**
- 普通访客无感知，即使直接访问受保护 URL 也会被密码墙拦截。
- 无传统注册登录系统，不引入用户表。

## 四、非功能需求

- **响应式**：移动端优先，所有页面适配手机。
- **暗色模式**：支持系统级切换及手动切换。
- **性能**：Lighthouse 评分 > 90，图片懒加载，列表分页。
- **可访问性**：满足 WCAG AA，键盘可操作。
- **安全与隐私**：严格脱敏，AI 对话防滥用，管理端密码保护。
- **AI 成本可控**：对话限流、每日上限、本地 Q&A 匹配优先。

## 六、核心数据模型概述

完整设计见 `DATABASE.md`，此处仅列出关键实体：

- **Entry**：内容主表（标题、类型、状态、时间、AI 分析结果等）。
- **TextContent**：分离存储的 Markdown 正文。
- **EntryModule**：模块分发（一条内容可出现在简历、时间轴、项目等多个模块）。
- **WorkExperience / Project / Knowledge / Travel / Lifestyle**：各类型专属子表。
- **Category**：分类树（用于知识库，后续可扩展）。
- **Tag**：标签多对多（MVP 可用 JSON 数组简化）。

知识库条目为特殊的 `knowledge_link` 类型，存储外链。

## 七、AI 产品设计

- **导游模式**：首次主动引导，再次访问气泡提醒，页面感知上下文。
- **混合问答**：本地 Q&A 优先，大模型兜底，成本极低。
- **批量智能导入**：自动拆分多段经历，提取字段，用户可编辑、选择模块后批量入库。
- **边界安全**：无关问题幽默拒绝，始终扮演分身角色。

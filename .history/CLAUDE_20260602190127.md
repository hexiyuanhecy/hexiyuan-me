## 六、数据库设计（多表关联 + Prisma）

数据库架构完全遵循项目根目录下的 `DATABASE.md`，采用 **核心主表 + 类型子表 + 模块分发表** 的多表关联方案。

### 6.1 技术环境
- 数据库：**SQLite**（开发环境，文件 `prisma/dev.db`）
- ORM：**Prisma 7.8.0**
- 生产环境可平滑迁移至 PostgreSQL（配合 Turso / Vercel Postgres）

### 6.2 核心表结构
- **Entry**：内容主表，存储标题、slug、类型、状态、摘要、事件时间、AI 分析等通用字段
- **TextContent**：长文本分离存储，关联 Entry
- **EntryModule**：模块分发关联表，一条内容可同时出现在多个前台模块（如 `resume`, `timeline`, `projects`）
- **WorkExperience / Project / Knowledge / Travel / Food / Daily**：各类型子表，存储专属字段
- **Category**：树状分类（可空，MVP 可暂不用）
- **Tag / EntryTag**：标签多对多（MVP 可在子表中用 JSON 数组代替）

### 6.3 枚举定义（与 `DATABASE.md` 完全一致）
- `EntryType`: `work_experience`, `project`, `knowledge`, `blog`, `travel`, `food`, `daily`, `other`
- `EntryStatus`: `draft`, `published`, `archived`
- `ContentType`: `markdown`, `rich_text`, `image`, `video`, `audio`, `file`
- `ModuleName`: `resume`, `timeline`, `projects`, `knowledge`, `blog`, `travel`, `lifestyle`, `games`

### 6.4 开发要求
- 所有数据操作必须基于 Prisma Client，严禁直接拼接 SQL。
- 时间轴查询统一使用 `COALESCE(occurred_at, created_at)` 排序。
- 导入功能需同步写入 Entry + 对应子表 + EntryModule，并用事务保证一致性。
- `DATABASE.md` 为权威参考，若本文件与它冲突，以 `DATABASE.md` 为准。

## 七、智能导入功能设计

### 用户流程
1. 访问 `/import`（需密码验证，JWT 短期会话）。
2. 提供"模板填写"和"自由粘贴"两个选项卡：
   - **工作经历模板**：
     ```
     公司：[公司名]
     职位：[职位]
     时间：[开始-结束，如 2021.03-2023.07]
     描述：[工作内容与成果]
     技术栈：[逗号分隔]
     ```
   - **项目经历模板**：
     ```
     项目名称：[项目名]
     简介：[一句话简介]
     技术栈：[逗号分隔]
     链接：(可选)
     亮点：[列表]
     ```
3. 提交文本 → `/api/import` 调用 DeepSeek 分析，返回：
   - `type`（内容类型）
   - `fields`（提取的字段 JSON）
   - `suggested_modules`（建议勾选的模块数组）
4. 前端展示**模块多选**，供用户勾选目标模块：
   - 若 `type === "work_experience"`，**"简历时间轴"自动勾选且不可取消**。
   - 其余模块根据建议预选，可手动增删。
5. 预览编辑表单：展示所有识别字段，允许用户任意修改。
6. 点击"确认入库" → `/api/import/confirm` → 存入数据库（同步写入 Entry + 子表 + EntryModule）。
7. 前端反馈："已添加至：简历时间轴、项目作品集"，并提供跳转链接。

## 八、权限控制（轻量方案）

- 保护页面：`/import` 和 `/admin`。
- 环境变量 `IMPORT_PASSWORD` 存储明文密码。
- 访问时弹出密码输入框（仅密码字段），前端发送 `POST /api/verify`。
- 后端校验密码后签发 JWT（有效期 24 小时），前端存入 `localStorage`。
- 后续所有请求在 `Authorization: Bearer <token>` 头部携带，后端中间件验证。
- 普通访客完全无感知，即便直接访问 URL 也会被密码墙拦截。

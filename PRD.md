**版本**：v2.0  
**最后更新**：2026-06-02  
**作者**：何西元（AI 辅助生成）  
**状态**：MVP 待开发

---

## 一、产品愿景

打造一个 **全栈、智能、有温度** 的个人数字花园，既是用 AI 武装的求职利器，也是个人知识资产与生活经历的立体展示。让面试官或访客通过 **右下角 AI 悬浮机器人** 随时与“我”对话，通过交互式时间轴一览成长脉络，并通过智能导入持续丰富内容。

---

## 二、目标用户

| 用户类型 | 核心需求 |
| :--- | :--- |
| **面试官 / HR** | 快速了解技术能力、项目经验，检验真实性和沟通能力 |
| **技术同好** | 浏览技术文章、学习笔记，进行技术交流 |
| **朋友或潜在合作者** | 了解个人兴趣、价值观、生活面 |

## 三、功能需求

### 3.1 MVP 核心功能（V1.0）

#### F1. 首页
- 个人简介、技能标签云、三大板块入口卡片（专业经历、成长时间轴、知识库）。
- 全局导航栏 + 移动端汉堡菜单。

#### F2. 专业板块
- **简历时间轴 `/resume`**：垂直时间轴，按年份筛选，数据动态加载。
- **项目作品集 `/projects`**：卡片网格，每张卡片含技术栈标签和链接。
- **项目详情 `/projects/[slug]`**：Markdown 渲染，从数据库读取。
- **技能雷达图**：CSS/SVG 实现，数据可配置。

#### F3. 前端知识库 `/knowledge`
- 文章列表，支持分类筛选。
- 文章详情 `/knowledge/[slug]`：MDX 渲染，含目录导航（TOC）。

#### F4. 成长时间轴 `/timeline`
- 混合展示工作、项目、学习、旅行等所有人生节点。
- 按事件发生日期倒序排列（未填日期则使用录入时间）。
- 每个节点点击可查看详情。

#### F5. 全局 AI 数字分身（右下角悬浮机器人）
- 全站右下角固定悬浮按钮，点击展开聊天窗口。
- 流式对话、打字机效果、自动滚动。
- 系统提示词注入“何西元”脱敏人设，支持面试模拟。
- 成本控制：前端频率限制、每日全站上限、高频问题缓存、管理员可关闭。

#### F6. 智能导入 `/import`（管理员专属）
- **文本输入 + 模板辅助**：提供工作经历、项目经历两种模板。
- **AI 分析**：识别内容类型并提取结构化字段。
- **模块选择**：用户勾选导入到哪些模块（简历时间轴、项目、知识库等）；工作经历类强制选入时间轴。
- **预览编辑**：可修改所有识别字段。
- **确认入库**：写入数据库，自动分发到所选模块（Entry + 子表 + EntryModule）。

#### F7. 管理后台 `/admin`（管理员专属）
- **列表展示**：所有条目的列表，支持搜索和筛选。
- **编辑功能**：编辑表单与导入预览一致，可修改所有字段。
- **删除功能**：一键删除，带二次确认弹窗防止误删。
- 权限控制：JWT 短期会话（24小时）。

#### F8. 权限控制
- `/import` 与 `/admin` 受密码保护（环境变量 + JWT 短期会话）。
- 无传统登录注册，访客无感知。

## 四、非功能需求

- **响应式**：移动端优先，所有页面适配手机。
- **暗色模式**：支持系统级切换及手动切换。
- **性能**：Lighthouse 评分 > 90，图片优化、静态生成。
- **可访问性**：满足 WCAG AA，键盘可操作。
- **安全与隐私**：严格脱敏，禁止暴露真实个人信息；AI 对话防滥用。
- **AI 成本可控**：单日对话限额、前端频率限制、高频问题缓存。

## 六、核心数据模型

所有内容采用 **多表关联架构**，完整定义见项目根目录下的 `DATABASE.md`，此处列出产品层面的关键实体与关系。

### 6.1 内容主实体（Entry）
- **通用属性**：标题、唯一标识 (slug)、类型、状态（草稿/已发布/归档）、摘要、事件发生时间、AI 分析结果。
- **正文**：长文本分离存储在 `TextContent` 表，支持 Markdown 或富文本。
- **模块分发**：通过 `EntryModule` 表将一条内容关联到一个或多个前台模块（如简历、时间轴、项目）。

### 6.2 类型专属实体（子表）
- **工作经历**：公司、职位、起止时间、技术栈。
- **项目经历**：项目名、描述、技术栈、链接、亮点列表。
- **知识库**：分类、标签。
- **旅行**：目的地、旅行日期、标签。
- **吃喝玩乐 / 日常**：分类、标签（可逐步扩展）。

### 6.3 分类与标签
- **分类树 (Category)**：支持无限层级，用于知识库和博客的组织。
- **标签 (Tag)**：多对多关联，可在 MVP 阶段先用 JSON 数组简化。

### 6.4 时空与 AI 属性
- **时空字段**：`occurred_at` 保证时间轴统一排序；`location` 预留地理位置（后期可用于地图可视化）。
- **AI 字段**：`ai_analysis` 存储导入时的 AI 分类及提取结果；`ai_vector` 预留语义搜索向量（V3 启用）。

## 七、AI 产品设计

- **数字分身**：基于脱敏经历的系统提示词，支持上下文对话、面试模拟。
- **智能导入**：文本分类 + 字段抽取，用户可干预并选择目标模块。
- **成本与安全**：严格限制调用频次和日总量，保护 API 额度不被恶意消耗。
